The Sauce to Create a Browser Support for VoidWeb: A VoidWeb Browser in a Nutshell

1. Introduction

The Void Web is a network that uses the vd:// addressing scheme, with specific protocols and APIs for processing URLs and navigation. This documentation explores the functioning of Carm and Carm Browser, the Void Web navigation standards, and the importance of ErebusAPI for DNS resolution. Understanding these components is crucial for developing and maintaining a browser that supports the Void Web, ensuring seamless navigation and content delivery.

2. Carm vs Carm Browser

Carm is an algorithm that processes and interprets URLs in the Void Web, while Carm Browser is an application that uses Carm to navigate these URLs. While Carm manages the logic for retrieving IPs and GitHub links for Void Web URLs, Carm Browser displays the content returned to the user. This separation of concerns allows for a modular approach to Void Web navigation, where Carm handles the backend logic and Carm Browser focuses on the user interface and experience.

3. IP Returned: IP vs GitHub

Carm uses the ErebusAPI to determine the destination of a URL. The ErebusAPI responds with a value based on the resource type:

  • IP Returned IP: If the URL is an IP, Carm directs Carm Browser to the specific IP returned by ErebusAPI. This allows for direct access to servers on the Void Web, providing a fast and efficient way to retrieve content.
  • IP Returned GitHub: If the URL is a GitHub repository, Carm Browser accesses the content in the repository as returned. This enables the use of GitHub as a content delivery network for Void Web applications, leveraging its robust infrastructure and version control capabilities.

4. GitHub Logic

When ErebusAPI returns a GitHub repository URL, Carm must access the repository structure, starting with the index.html file as the required entry point. This structure allows Carm Browser to display the project's or application's home page. By adhering to this convention, developers can ensure that their projects are easily accessible and navigable within the Void Web ecosystem.

5. IP Logic

If ErebusAPI returns an IP, Carm redirects Carm Browser directly to this address. This IP represents an active server on the Void Web, which is directly accessible without intermediaries. This direct access model is ideal for real-time applications and services that require low latency and high performance.

6. Creating Custom Functions (Example: GitLab)

Although Carm natively supports GitHub, you can create custom functions for integration with other services, such as GitLab. To do this, adapt Carm's logic to access the GitLab repository as needed. However, we recommend using the default systems (GitHub and IPs) supported by the Void Web for maximum compatibility. Custom functions should be thoroughly tested to ensure they adhere to Void Web standards and provide a seamless user experience.

7. Void Web Navigation Standards

The Void Web has navigation standards that must be followed to ensure compatibility:

  • Initial File: The browser must always request index.html as the primary entry point. This ensures a consistent starting point for all Void Web applications, simplifying navigation and content delivery.
  • ErebusAPI: All Void Web URLs must pass through ErebusAPI for DNS resolution and proper routing. This centralized approach to DNS management enhances security and reliability, ensuring that all URLs are correctly resolved and routed to their intended destinations.

8. ErebusAPI: DNS Management

The ErebusAPI is the DNS resolution API of the Void Web. When a vd://domain.tld URL is requested, Carm queries ErebusAPI using the format http://localhost:4567/<dn>/<tld>/ip. The API returns an IP or a GitHub link for the Browser to display the appropriate content. This API is a critical component of the Void Web, providing the necessary infrastructure for URL resolution and content delivery.

9. Advantages of Void Web Standards

Void Web standards provide several key advantages:

  • Consistency and Compatibility: Ensures a uniform experience across any browser or tool using the Void Web. By adhering to these standards, developers can create applications that are compatible with a wide range of devices and platforms.
  • Security: ErebusAPI controls access to URLs, improving network security. This centralized approach to DNS management helps prevent unauthorized access and ensures that all URLs are correctly resolved and routed.
  • Ease of Integration: Ensures that content is displayed correctly by following the standard file order. This simplifies the development process and ensures that applications are easy to navigate and use.
